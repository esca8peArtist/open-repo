## Reallocation Triggers (What If)

### If researcher becomes unavailable mid-Phase 2:

**Trigger**: Researcher ill / resignation / family emergency during active research

**Action (within 24 hours)**:
1. Identify which domain affected
2. If Domain 59 (highest urgency): pull researcher from Domain 48 or 49-50 pool (negotiate delivery delay)
3. If Domain 51: activate Tier 2 recruitment immediately (CLC if primary unavailable; Issue One secondary)
4. If Domain 49-50: delay that domain 1-2 weeks; reallocate hours to Domain 48 or 51
5. If Domain 48: extend timeline to August 15 (lowest urgency)
6. Notify user within 24 hours; adjust publication timeline if needed

**Capacity shift example**:
- Domain 59 researcher unavailable mid-June → pull 10h from Domain 48 (July start) → extend Domain 48 to Aug 15
- Domain 51 researcher unavailable → activate 2 Tier 2 contacts immediately → reduce hours 40→30h (more leverage, less grunt work)

### If Phase 2 decision slips past target date:

**Trigger**: Day 7 decision delayed (user doesn't provide input; diagnostic extends past June 25)

**Action**:
- Domain 59 can slip max 2 days (Senate Finance deadline is hard: June 25-30)
- Domain 51 can slip max 5 days (CA Fair Elections Act deadline July 1 has ~5 day buffer)
- Domains 48, 49-50, 57 not impacted (July 1+ activations have flexibility)
- If Day 7 decision occurs June 27+, activate STRONG path immediately (avoid incremental decision-making)

### If Phase 2 research quality is substandard:

**Trigger**: First draft review shows quality <80% (missing sources, weak argumentation, inconsistent depth)

**Action**:
- If quality issue in 1-2 domains: extend timeline for those domains (+1-2 weeks for revision)
- If quality issue across all domains: pause Phase 2, conduct researcher training session, restart with adjusted expectations
- Reallocate hours from deferred domains (57, other backlog) to quality remediation
- Example: Domain 59 draft is weak → extend June 30 deadline to July 10 → use 5h from Domain 57 pre-research for revision
