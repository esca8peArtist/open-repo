# Phase 4 Activation Router

> **Read this document first on June 1.**
> Input: May 31 CHECKIN decision (publication option + Phase 6 domain selections + user availability).
> Output: Which activation template to use and whether any modifications apply.
> Time to complete: 5–10 minutes.

---

## Step 1: Read the May 31 Decision Inputs

Open the May 31 CHECKIN and locate three pieces of information:

**A. Phase 5 publication timeline choice**
Did the user select Option A (Wave 1+2 June 1–5, Wave 3 June 30), Option B (unified June 15), or Option C (rolling modular, 1 module/week)?

If the May 31 CHECKIN does not record a clear selection: proceed to the Fallback section at the bottom of this document.

**B. Phase 6 domain selections**
Which Phase 6 domains did the user approve? Options from the Phase 6 decision support package:
- Community Economic Resilience
- Intergenerational Skills Development
- Governance Scaling
- None selected (Phase 6 deferred)
- Multiple domains selected

This affects Phase 4 scope because Phase 6 domains that overlap with Phase 4 governance or household scaling topics require Phase 4 to establish a stronger foundation before Phase 6 research begins.

**C. User availability**
Is the user available for concentrated work (4–6 hours/day), part-time (1–2 hours/day), or episodic (blocks of time, unpredictable schedule)?

This affects which activation template modifications apply.

## Step 2: Route to the Correct Template

### If Option A selected:

Use **PHASE_4_ACTIVATION_OPTION_A.md** in this directory.

**Phase 6 modifications:**
- If Governance Scaling is selected as a Phase 6 domain: during the governance synthesis workshop (Deliverable 1), document all governance architecture decisions explicitly — they are the foundation for Phase 6 Governance Scaling research. Add 15 minutes to Hour 2 of the workshop to record decision rationale (not just the decision itself).
- If Community Economic Resilience is selected: during the Zone 5 adaptation guide (Deliverable 3), add one paragraph on cooperative economic infrastructure in Zone 5 (rural electric cooperatives, farmer cooperatives). This is the Phase 4 bridge content Phase 6 will build on.
- If no Phase 6 domains are selected: run Deliverable 1–3 as written, no modifications.

**Availability modifications:**
- Full-time availability: run the governance synthesis workshop June 1–2 (2 intensive days) instead of June 1–3 (3 moderate days). Complete all three deliverables by June 2 evening. June 3–4 are buffer days.
- Part-time availability: run the governance synthesis workshop June 1–4 (4 half-days). June 5 is the completion gate (same as written). No other modifications needed.
- Episodic availability: the governance synthesis workshop requires 3 consecutive sessions with the same participants. If scheduling is unpredictable, schedule all three workshop sessions before June 1. If that is not possible, Option A is at risk — consider switching to Option C (rolling model tolerates episodic coordination better).

---

### If Option B selected:

Use **PHASE_4_ACTIVATION_OPTION_B.md** in this directory.

**Phase 6 modifications:**
- If Governance Scaling is selected: the individual-scale deepening in Week 1 (June 1–7) should include one additional methodology — **governance legitimacy assessment** (how does a community evaluate whether its governance is producing decisions that members accept as legitimate, not just formally correct?). This methodology does not exist in Phase 3 or Phase 5. Week 1 produces a first draft; Phase 6 Governance Scaling refines it.
- If Community Economic Resilience is selected: the household scaling synthesis in Week 2 (June 8–10) should include one additional dimension — **household economic resilience** (what financial resources does the household have, and how does that affect its capacity to participate in community coordination?). This is the bridge from Phase 4 household scaling to Phase 6 Economic Resilience.
- If Skills Development is selected: during the household scaling synthesis, document which skills are present in the household arrangement and which are gaps. This skills inventory becomes Phase 6 Skills Development source material.
- If no Phase 6 domains are selected: run Week 1 and Week 2 as written, no modifications.

**Availability modifications:**
- Full-time availability: Option B as written is optimized for full-time availability. No modifications.
- Part-time availability: Option B is the most resource-intensive option. If availability is genuinely part-time (1–2 hours/day), extend Week 2 by 3 days (governance charter adoption moves to June 15 instead of June 12; publication moves to June 18). Notify the user that the June 15 date may slip to June 18. Use the 15-minute daily standup to identify blockers early.
- Episodic availability: Option B requires consistent week-over-week progress. Episodic availability is not a good fit for Option B's 14-day integrated ramp. If availability is genuinely episodic, recommend switching to Option A (3-day core + monitoring) or Option C (tolerable for episodic coordination).

### If Option C selected:

Use **PHASE_4_ACTIVATION_OPTION_C.md** in this directory.

**Phase 6 modifications:**
- If Governance Scaling is selected: the Week 7 final charter documentation (after Phase 5 is complete) should include a governance evolution narrative — a 3–4 paragraph summary of how governance changed each week and why. This narrative is Phase 6 Governance Scaling source material. It cannot be reconstructed after the fact; it must be written at Week 7.
- If Community Economic Resilience is selected: in the Module Pairing Matrix, Week 6 (Fuel Production + Educational Governance) should include one additional Phase 4 task — document the community's existing cooperative economic relationships (rural electric cooperative membership, farmer cooperative access, mutual aid networks). One page. This is the Phase 4 bridge for Phase 6 Economic Resilience.
- If Skills Development is selected: add a standing item to each Thursday assembly: "What skill did we use or need this week that we don't have?" This takes 5 minutes. By Week 6, the community has a 6-week skills demand log that is Phase 6 Skills Development source material.
- If no Phase 6 domains are selected: run the rolling template as written, no modifications.

**Availability modifications:**
- Full-time availability: run the Thursday assemblies plus Tuesday and Wednesday preparation as written. Consider compressing the pre-Week-1 governance skeleton (May 29) into a same-day session June 1 to start immediately.
- Part-time availability: Option C is optimized for part-time availability. The rolling structure distributes work across 8 weeks with no single high-demand period. No modifications needed.
- Episodic availability: Option C can work with episodic availability if Thursday assembly attendance is protected. If Thursday evenings are consistently available but other days are not, Option C works — the Tuesday and Wednesday preparation tasks can be compressed to 30 minutes on Thursday afternoon.

## Fallback: Decision Not Made by June 1

If the May 31 CHECKIN does not record a publication option selection, or if it records ambiguity ("leaning toward A but considering B"), activate the fallback:

**Default: Option B template with 2-week buffer.**

Rationale: Option B is the most reversible choice. If the user later decides on Option A, the governance workshop (identical in A and B) is already complete, and the Phase 5 editorial work done in Days 1–7 is not wasted. If the user later decides on Option C, the governance charter adopted under Option B provides the foundation that Option C's pre-Week-1 governance skeleton requires.

**Fallback modifications:**
- Wave 1+2 publication target: June 15 instead of June 1 (2-week buffer absorbs the decision delay)
- Governance workshop: June 1–3 (unchanged)
- Phase 5 editorial work: begins June 1 (unchanged)
- Everything else: proceeds per Option B template

**Fallback record in ORCHESTRATOR_STATE.md:**
"June 1 — Phase 5 publication option not recorded at May 31 CHECKIN. Defaulting to Option B with 2-week buffer (June 15 publication target). User must confirm or override by June 5."

The fallback requires the user to confirm or override by June 5. If the user confirms Option B by June 5: proceed per template. If the user selects a different option by June 5: switch templates. The governance workshop (June 1–3) is reusable regardless of which option is confirmed.

If no user confirmation is received by June 5: continue Option B execution. The 2-week buffer (to June 15) gives adequate time for a decision to arrive without derailing the Phase 4 or Phase 5 timeline.
