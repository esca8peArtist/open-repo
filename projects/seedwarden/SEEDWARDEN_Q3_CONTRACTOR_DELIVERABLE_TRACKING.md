## 3. Quality Gate Procedures — Pass/Fail Checklists

### 3.1 Photographer Quality Gates

**Gate applies to**: Each session delivery (Sessions 1, 2, 3). Run gate within 48 hours of receipt.

**Gate 1 — Session delivery (5 images)**

Run this checklist immediately upon receipt. If all items pass, move to "Approved — send M2 payment" (or M3, M4 for Sessions 2-3). If any item fails, send revision email (Template 3B in PHASE_3_CONTRACTOR_COMMUNICATION_TEMPLATES.md) within 24 hours.

- [ ] All 5 images delivered (flat file format: JPG or PNG, no TIFF)
- [ ] File naming convention followed: `[bundle-slug]-[type]-[sequence].jpg` (e.g., `respiratory-flatlay-topdown.jpg`)
- [ ] Minimum resolution 2000×2000px confirmed on each image (check file properties or Lightroom metadata)
- [ ] sRGB color space verified on all 5 images (check image properties; no Adobe RGB or other color spaces)
- [ ] White balance consistent across the 5-image set (compare text readability and skin tone warmth across all 5; no blue/orange cast drift)
- [ ] No harsh shadows obscuring herbs or guide cover text mockup (visual inspection; shadows should be soft and diffuse)
- [ ] Commercial license clause referenced in photographer contract (work-for-hire; no additional license required from photographer for Etsy commercial use)
- [ ] File transfer method verified (Google Drive, Dropbox, Wetransfer, or email link; files not corrupted in transfer)

**Pass criteria**: All 8 items checked. Session approved.

**Fail criteria**: Any 1 item unchecked. Send revision email with specific item(s) requiring correction. Allow 3-day revision window (revision due 3 days after revision email sent).

**Approved action**: Send praise email (Template 3A in PHASE_3_CONTRACTOR_COMMUNICATION_TEMPLATES.md) within 48 hours of gate pass. Include: specific image praise (e.g., "The flat-lay on the linen surface with the dried elderberry branch has the exact warmth this series needed"). Release Milestone 2 payment same day as approval email.

### 3.2 Writer Quality Gates

**Gate applies to**: Each bundle first draft, revision round, and final draft. Run gate within 72 hours of receipt.

**Gate 1 — First draft review**

| Criterion | Pass | Fail | Notes |
|---|---|---|---|
| Word count within tolerance | 3,000–3,800 words ±10% of target | Outside ±10% range | Allow typo/placeholder words; measure final copy only. Respiratory 3,600w ±360w, Immunity 3,800w ±380w, Digestive 3,600w ±360w. |
| All species include Latin binomial on first mention | Every species: "Common Name (*Genus species*)" | Any species missing Latin binomial | Non-negotiable. Checks both scientific accuracy and Seedwarden brand standard. |
| FTC compliance: no unqualified therapeutic claims | Phrases: "Studies suggest…", "May support…", "Traditionally used for…", "Evidence suggests…" framed as evidence-tier | Phrases: "Prevents", "Cures", "Treats" without evidence-tier framing | Text examples: "Elderberry prevents colds" = FAIL. "Studies suggest elderberry may reduce cold duration" = PASS. |
| CITES sidebar present (Immunity bundle only) | Goldenseal section includes 200-word CITES sidebar verbatim text from PHASE_3_CITES_SIDEBAR_TEMPLATE.md | Sidebar missing, abbreviated, or paraphrased | Immunity bundle only. Sidebar is non-negotiable word-for-word language to ensure FTC/CITES compliance. |
| Contraindication panels cite primary source database | Every contraindication cites: NatMed Pro, MSK Integrative Medicine, Wolters Kluwer (or equivalent peer-reviewed database) | Contraindications without citation OR cite to "herbalist knowledge" or non-peer-reviewed source | Primary source attribution required. Acceptable databases: NatMed Pro, MSK Integrative Medicine, Micromedex, UpToDate, or published herbal reference (Botanical Safety Handbook, PDR for Herbal Medicines). |

## 5. Contingency Procedures — Late Deliverable Activation

**Trigger conditions**: Deliverable not received by due date at 5pm UTC / 1pm ET.

**Action on due date (same day, 6pm UTC / 2pm ET)**:

Send check-in message via contractor's platform (Upwork, email, direct message):

> Hi [NAME],
>
> [DELIVERABLE] was due today by [DUE TIME]. I don't see it in my inbox yet. Can you confirm status? If delivery is delayed, please let me know expected delivery date by end of day today so I can adjust the schedule.
>
> If there is a blocker I can help clear, let me know.
>
> [YOUR NAME]
> Seedwarden

**Contractor response window**: 24 hours (due date + 24 hours = Day 1, 5pm UTC)

**Outcome**:
- If deliverable received within 24 hours: proceed to checkpoint process (Section 4 above), accept 1-day delay, no escalation
- If contractor confirms delay and provides new delivery date within 48 hours: accept new date, log in PHASE_3_EXECUTION_LOG.md as "DELAYED — [DATE]", proceed with revised timeline
- If no response and no deliverable by Day 1 EOD: move to Level 2
