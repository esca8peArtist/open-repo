## Section 1: Content Scope — Species Priority Matrix

### How Priority Was Set

Four filters applied in order:

1. **Guide coverage gap**: Phase 1 has 18 wild edibles and a partial native plants regional guide. Any species with no guide entry gets highest weight — it is net-new coverage, not improvement of existing content.
2. **Seasonal window May–October**: Species that are past their peak by the time guide content would publish (July–August) have reduced priority for the initial launch batch. They slot into later seasonal updates.
3. **Regional distribution breadth**: Species present in 4+ US regions are prioritized over regional endemics. Broad distribution means a larger addressable buyer pool.
4. **Photo-to-guide dependency**: Species where the photography logistics plan already commits a field session can move directly into writing after May 30. Species with no scheduled photo session require sourced Wikimedia/iNaturalist images and are flagged accordingly.

## Section 2: Guide Writing Pipeline

### The Pipeline in Brief

Every guide moves through six stages. Stages 1–3 happen in the field. Stages 4–6 happen at the desk. Do not skip ahead — specifically, do not begin writing Stage 5 until Stage 4 is confirmed.

```
Stage 1: Field data capture (on-site, per-specimen)
Stage 2: Photo cull and species ID verification (within 72 hours of session)
Stage 3: Habitat assessment transcription (within 24 hours of session)
Stage 4: Photo-to-section mapping (before writing begins)
Stage 5: Guide draft — section by section
Stage 6: QA pass and Canva integration
```

---

### Stage 1: Field Data Capture

Use the Specimen Data Capture Template in `PHASE_2_PHOTOGRAPHY_LOGISTICS.md` Section 5 for every species documented. Minimum required before a guide can be written:

- Species ID confidence marked High (or verified to High through iNaturalist consensus or regional key)
- Phenological stage recorded (determines which "season" section applies)
- Lookalike species noted or marked absent at site
- Habitat assessment form completed for the site (Section 4 of photography logistics)

**Do not write the guide if species ID confidence is Medium or Low.** Write a placeholder entry in the guide draft with "PENDING VERIFICATION" and return to it after iNaturalist consensus or expert check.

---

### Stage 2: Photo Cull and ID Verification

Within 72 hours of the field session:

1. Apply the culling standard from `PHASE_2_PHOTOGRAPHY_LOGISTICS.md` Section 6 — discard focus failures, frames without scale bars (where required), and frames where toxic lookalike could be misread.
2. Flag any frame with ID confidence below High with a yellow label.
3. For any species where ID is flagged: submit to iNaturalist as an observation and wait for three independent ID confirmations at the species level before moving to writing. This typically takes 24–72 hours for common species.
4. Create the session folder and apply metadata before closing this stage.

---

### Stage 3: Habitat Assessment Transcription

Transcribe field notebook data to `projects/seedwarden/data/habitat-assessments/YYYY-MM-DD_[site-slug]-habitat.csv`. This takes 10–15 minutes per site. The data feeds directly into the "Where to Find It" section of each guide entry.

Map the assessment fields to guide sections:
- Elevation + aspect + moisture → "Where to Find It" body text
- Canopy + shrub species → companion plant associations
- Disturbance level + invasives → harvest ethics note
- Associated species list → "Often found near..." sentence in the guide

---

### Stage 4: Photo-to-Section Mapping

Before writing a single sentence of guide text, complete this mapping table for each species. This step takes 5–10 minutes and eliminates the most common production error: writing a section that has no photo to illustrate it.

| Photo on hand | Guide section it covers | Available? |
|---------------|------------------------|------------|
| Habitat context (3–5m) | "Where to Find It" inline image | Yes / No |
| Full plant habit + scale bar | Primary ID header image | Yes / No |
| Key ID feature closeup | "Key Identification Features" | Yes / No |
| Seasonal feature (flower/fruit) | "Season" + "Harvest Timing" | Yes / No |
| Lookalike comparison | "Important — Do Not Confuse With" callout box | Yes / No |
| Harvest method demonstration | "How to Harvest Ethically" | Yes / No (optional) |

If the habitat context photo is missing: write the "Where to Find It" section without an inline image and note it for the next field session.  
If the primary habit photo is missing: **do not publish this entry**. Wait for a verified photo before writing the identification section.

---

### Stage 5: Guide Draft — Section Order and Word Targets

Write each entry in section order. Do not jump between sections. The standard entry structure mirrors the Phase 1 native plants guide template exactly:

**Header line**: Common Name / *Scientific name* (italic)  
**Photo**: Primary habit photo with alt text in format: `[Common Name] ([Scientific name]) — [one-line description of what the photo shows]`

**Identification** (80–120 words)  
Lead with the single most reliable distinguishing feature. State it in the first sentence. Follow with supporting features in order of reliability. Close with the most important lookalike distinction, stated as a direct comparison. Do not bury the lookalike note at the end of a long paragraph — move it to the front if the lookalike is deadly.

**Where to Find It** (60–90 words)  
Ecosystem first, then specifics. Pull directly from habitat assessment data. Include elevation range if relevant, moisture/soil preference, and 1–2 companion species that help confirm the location. State regional distribution breadth: "Found across the eastern US from Maine to Georgia, most abundant in [specific conditions]."

**Season** (30–50 words)  
State the harvest window as a month range, not vague seasonal language. Note how this varies by latitude or elevation. Include which parts are edible in each stage. If a part is past edibility by late season, state it explicitly: "Leaves become unpalatable after flowering — harvest before bloom."

**Edible Parts** (20–40 words)  
List parts. Note which require preparation and which are edible raw. If any part is toxic, state it in this section with bold text: **Do not eat [part] — toxic at all stages.**

**Preparation** (60–100 words)  
2–4 concrete preparations, not vague suggestions. Include at least one simple method (raw or minimal cooking) and one cooked method. State cooking temperatures or times where relevant — "blanch 5 minutes" is more useful than "cook briefly." Include flavor description: "mild, slightly grassy" or "tart, similar to rhubarb."

**Safety** (30–60 words)  
State the most dangerous lookalike by name in bold. Give one or two diagnostic features that definitively separate safe from toxic. End with a clear action statement: "If in doubt, do not eat — [toxic species] is deadly and the consequence of misidentification is irreversible."

**Notes** (optional, 20–40 words)  
Harvest ethics, conservation status, or cultural significance if any of these applies. Ramps, Camas, Ginseng, and Goldenseal require this section. Invasive species entries should note that harvesting is encouraged.

---

### Stage 6: QA Pass and Canva Integration

Before placing content in Canva:

1. Read the draft aloud. Any sentence that requires more than one re-read to parse is too complex — simplify it.
2. Confirm every photo has alt text.
3. Confirm the lookalike is named by scientific name, not just common name (common names are inconsistent regionally).
4. Confirm the safety section is in the guide, not omitted.
5. Confirm photo attributions match the WORKLOG.md entry for that species.

Canva placement follows the photo-to-section map from Stage 4. Use the master template established on May 26 for the Appalachian Medicinals guides. The native plants / wild edibles template uses the same typeface and color system; the only difference is the section structure (medicinals guides include a Conservation Status section that wild edibles guides replace with a Harvest Ethics section).

## Section 3: Photo-to-Guide Integration

### From Photography Logistics to Guide Figures

The species priority matrix in `PHASE_2_PHOTOGRAPHY_LOGISTICS.md` Section 2 was built with guide content requirements in mind. Each photo angle in the field protocol maps directly to a guide figure. This table is the translation layer between the photography plan and the writing pipeline.

| Photography protocol angle | Guide figure type | Placement |
|---------------------------|------------------|-----------|
| Habitat wide (3–5m, no scale bar) | "Where to Find It" inline image | Below first paragraph of Where to Find It |
| Full plant habit (scale bar) | Primary ID header / thumbnail | Page header image; also used in Etsy listing slot 4 or 5 |
| Key ID feature closeup (macro, scale bar) | "Key Identification Features" callout | Alongside or below the Identification section |
| Seasonal feature (flower/fruit) | "Season" section inline | Alongside Season or Harvest Timing text |
| Lookalike comparison | "Important — Do Not Confuse With" callout box | At the end of Identification or in a prominent safety callout box |
| Harvest method demonstration | "How to Harvest Ethically" | Alongside the Notes section |

### Photo Quality Minimums for Guide Inclusion

A photo does not meet the guide standard if:
- Focus failure on the primary ID feature (stem hair line, leaf margin, flower structure)
- Toxic lookalike is visible in frame without a clear caption distinguishing it
- Species ID confidence was below High at time of capture and iNaturalist verification is not yet complete
- Exposure requires more than +1 stop of recovery in post-processing

For any Tier 1 species where field photography fails to meet these standards, use Wikimedia Commons CC0 or CC-BY as a fallback. Log the source and license in WORKLOG.md using the standard entry format from the photography logistics appendix before integrating the image into the guide.

## Section 4: Seasonal Content Strategy

### The Four Windows and What to Write for Each

Phase 2 content should not all be written at once and released flat. Seasonal relevance is a meaningful SEO and engagement signal. Structure the publication cadence around what the buyer is experiencing in the field.

**Window 1 — Spring Ephemerals (write May–June, publish June–July)**  
Species: Ramps, Ostrich Fern, Miner's Lettuce, Trout Lily, Solomon's Seal, Wild Ginger, Bloodroot (cultural/seed context), Chickweed (bolting), Wood Sorrel (continuous)  
Content angle: "These species are gone by the time most people notice them — here is how to find them before the window closes." Urgency framing works for spring ephemerals. Guides published in June for following-year reference accumulate search traffic over 12 months.  
Photo timing: All field photography must be complete by May 30; Trout Lily and Bloodroot bloom photos are missed — use Wikimedia for bloom stage, field photos for foliage and habitat context.

**Window 2 — Summer Perennials (write June–July, publish July–August)**  
Species: Cattail (multiple stages), Stinging Nettle (post-flower warning), Wild Bergamot (peak bloom), Elderberry (fruit stage approaching), Purslane (summer peak), Black Raspberry, Mulberry, Spiderwort  
Content angle: "What is harvestable right now." Summer buyers are active in the field; guides with current seasonal information convert better than generic reference content. Cattail pollen harvest is a July SEO opportunity.  
Photo timing: All photos available from May–June field sessions. Writing can begin July 1 with complete image sets.

**Window 3 — Fall Seed Maturation (write August–September, publish September–October)**  
Species: Hawthorn (ripe berries), Beach Plum (fruit stage — late summer in mid-Atlantic), Dandelion (seed head ID), Jerusalem Artichoke (tuber harvest), Wild Strawberry (runners and fall replanting), Amaranth (seed harvest)  
Content angle: "Harvesting seeds and fruit before frost." Fall content serves two audiences: foragers taking advantage of peak harvest and gardeners collecting native seeds for propagation. Both audiences are actively searching October–November.  
Photo timing: Most fruit-stage photography requires a return field session in September. Flag these in the photo planning notes. The guide can be written with placeholder [FRUIT PHOTO — September] and the photo inserted after the September session.

**Window 4 — Winter Structure and Habitat ID (write October–November, publish November–December)**  
Species: Queen Anne's Lace (seed head + distinguishing from Hemlock in dormant state), Wild Asparagus (fern stage locating for spring), Burdock (dormant taproot), Cattail (winter seed dispersal context)  
Content angle: "Finding and marking locations in winter so you are ready in spring." Winter identification of woody and persistent structures is a distinct skill set with dedicated buyer demand. This window is low-volume but high-differentiation — few competitors cover it well.  
Photo timing: Wikimedia sources sufficient for most winter-structure photography. No field session required.

## Section 5: Database and Cross-Reference Structure

### Species Data Structure

Each species in the database requires the following fields before a guide can be published. These map directly to guide sections and prevent incomplete entries from going live.

```
species_slug:         (e.g., "urtica-dioica")
common_name:          (primary)
common_name_alt:      (regional variants)
scientific_name:      
family:               
growth_form:          [annual / biennial / perennial / shrub / tree]
native_status:        [native / naturalized / invasive]
edibility_status:     [prime / edible-with-prep / medicinal-only / toxic / uncertain]
regions:              [NE / SE / MW / SW / NW / Rocky / Great Plains / Pacific / All]
ecosystem_primary:    
ecosystem_secondary:  
season_harvest_start: [month]
season_harvest_end:   [month]
photo_habit_file:     
photo_habitat_file:   
photo_id_feature_file:
photo_seasonal_file:  
photo_lookalike_file: [optional]
photo_source_all:     [field / wikimedia-cc0 / wikimedia-ccbysa / inaturalist-ccby]
lookalike_species:    [scientific name of primary dangerous lookalike]
lookalike_toxicity:   [toxic / non-toxic / unknown]
guide_status:         [not-started / draft / reviewed / published]
worklog_entry_date:   
```

Store this as a CSV at `projects/seedwarden/data/species-database.csv`. One row per species. This file is the single source of truth for guide status tracking across all phases.

### Geographic Cross-Reference

Each species entry includes a `regions` field using the nine-region system established in the Phase 1 native plants guide (NE / SE / MW / SW / NW / Rocky / Great Plains / Pacific / All). This field enables three uses:

1. **Regional guide variant creation**: If a species is present only in NE + SE, the guide entry can note "not found west of the Mississippi" rather than presenting false universality.
2. **Bundle packaging**: A "Northeast Foraging Bundle" can be assembled automatically from all species with NE in the regions field.
3. **Search term targeting**: Region-specific keywords ("foraging guide northeast," "wild edibles pacific northwest") can be inserted accurately into guide metadata.

For species with wide distribution but regional habitat variation (e.g., Stinging Nettle — present everywhere but grows in different moisture regimes), add a `regional_notes` column with brief habitat variation text. This text inserts directly into the guide's "Where to Find It" section.

### Ecosystem Integration

The companion plant guild data in `native-plants-guide-expansion.md` provides a cross-reference layer between individual species and their ecosystem context. When writing a guide for any species that appears as an anchor, support, or ground layer species in a guild, link both directions:

- In the species guide: "Often found alongside [companion species] in [ecosystem] — see the [guild name] companion planting section."
- In the companion planting guild: link species name back to the full species guide entry.

This cross-linking increases guide-to-guide navigation and increases session depth (time on product) when guides are delivered digitally.

The five ecosystem types from the photography logistics plan (desert/arid, riparian/wetland, oak woodland, coastal, temperate forest) map to a `habitat_tag` field in the species database. Use these tags to build ecosystem-specific browse pages or bundles.

## Section 6: Guide Writing Timeline and Publishing Cadence

### Immediate Post-May 30 Priority

The May 31 through June 7 writing sprint is the most important window. This is when field photography is freshest, notes are most detailed, and the May 30 Phase 2 launch has just gone live. Momentum matters.

**May 31–June 6: Six no-dependency guides**  
Write the six Wikimedia-sourced Tier 2 guides first (Purslane, Dock, Red Clover, Plantain, Lamb's Quarters update, Japanese Knotweed update). These have no photo processing dependency. Writing time: approximately 90 minutes per guide at the target word counts. Six guides in seven days = one guide per day. This is achievable as a daily 90-minute work block.

**June 7–June 21: Field-photo Tier 1 batch writing**  
Write one to two guides per day from the Tier 1 list as each species' photo set clears processing. The bottleneck is photo culling and iNaturalist verification for any ID-uncertain frames. Running photo processing and writing in parallel (process morning session photos while writing the previous session's guides in the afternoon) is the most time-efficient pattern.

Target daily output: 1 guide per session, 4–5 sessions per week, guides published as completed rather than held for a batch release. Publishing as-you-go improves SEO indexing speed and gives you early conversion data on which guide formats and species topics perform best.

### Weekly Cadence: Recommended Rhythm

**Bi-weekly publishing** (every two weeks) is the minimum for maintaining search indexing momentum. Weekly publishing is the target if writing bandwidth allows.

The recommended pattern is:
- **Tuesday** (or any consistent weekday): publish new guide(s) to Etsy as new listings and/or update the existing native plants guide PDF with the new species
- **Friday**: share the newly published guide on social media (Pinterest pin, Instagram reel, or TikTok if video was captured in the field)

Do not attempt to hold all 20 Tier 1 guides for a single big-batch release. Early guides generate reviews, views, and search history that compound before the later guides launch.
