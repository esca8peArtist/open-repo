## Branch Determination Logic

**Apply exactly one threshold. Use the first that matches.**

```
IF: Score 3+ replies ≥ 5 AND ≥ 3 domains showing at least 1 Score 3+ reply
  → STRONG BRANCH

IF: Score 3+ replies 2-4 OR (Score 3+ replies = 1 AND that reply is Score 4-5)
  OR 1-2 domains with ≥30% reply rate
  → MODERATE BRANCH

IF: Score 3+ replies 0-1 (none at Score 4-5) AND overall reply rate < 30%
  → DRY BRANCH

IF: Score 3+ replies = 0 AND delivery rate < 80% (3+ hard bounces)
  → DELIVERY DIAGNOSTIC BRANCH (see end of document)
```

## DELIVERY DIAGNOSTIC BRANCH

**Threshold**: Score 3+ replies = 0 AND delivery rate < 80% (3+ hard bounces confirmed from send logs).

**What this means**: The problem may be delivery infrastructure, not messaging or audience alignment. Run the diagnostic before concluding the campaign is DRY.

**Immediate actions**:

1. [ ] Review DOMAIN_51_DISTRIBUTION_EXECUTION_LOG.md, domain-59-send-log-june1.md, and DOMAIN_48_DISTRIBUTION_SEND_LOG_TEMPLATE.md for bounce-back records
2. [ ] Identify which specific addresses bounced hard
3. [ ] Find alternate contact paths for bounced organizations (see Scenario A in DOMAIN_49_CONTINGENCY_DECISION_TREE.md for the alternate routing protocol)
4. [ ] Re-verify Gist URLs are live: confirm HTTP 200 for each Gist before assuming delivery issues
5. [ ] Re-send to recovered bounce addresses using alternate contact paths

After recovery, recalculate delivery rate and re-enter the main decision tree (STRONG/MODERATE/DRY) with the full corrected dataset.

**If delivery rate recovers to ≥80%**: Apply main branch logic.
**If delivery rate cannot recover above 80%**: Treat as DRY (proceed with DRY branch) and add infrastructure check (are the Gist URLs being flagged by spam filters? Test by sending from a different email account).
