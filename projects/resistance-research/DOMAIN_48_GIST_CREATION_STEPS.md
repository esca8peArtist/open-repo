## Step 5 — Set Domain 48 Gist Visibility

**Recommended setting: Secret Gist**

On the bottom-right of the Gist creation form, locate the dropdown button and set it to **"Create secret gist"**.

- **Secret**: Anyone with the URL can view; does not appear in public Gist search or profile listings. Appropriate for targeted professional distribution to named organizational contacts.
- **Public**: Appears in Gist search and is visible on your profile. Use if you intend broad organic discovery (appropriate for Tier C/D social distribution, not required for Tier A/B professional outreach).

For Domain 48's Wave 1 and Wave 2 professional distribution (Sentencing Project, PPI, Brennan Center, Worth Rises, CLC, M4BL), Secret is appropriate. The Secret Gist URL is fully shareable — anyone clicking the link can read the document without authentication. Secret only means it does not appear in public search.

## Troubleshooting

**Problem**: Gist shows as plain text (no formatting)
**Solution**: Verify filename ends in `.md`. Edit the Gist (pencil icon), correct the filename to `domain-48-criminal-justice-civic-exclusion-architecture.md`, save. GitHub Gist re-renders automatically after filename correction.

**Problem**: Gist text appears truncated (Sources section missing)
**Solution**: Edit the Gist, scroll to the bottom of the editor, verify the full source section is present (41+ citations). If truncated, paste the missing text at the end and save.

**Problem**: URL requires login when tested in incognito
**Solution**: Verify the Gist was created as "Secret" or "Public" (not accidentally saved as a draft). Delete and recreate with explicit "Secret" selection if needed. Secret Gists are publicly accessible to anyone with the URL — "secret" means not discoverable via search, not private.

**Problem**: Source URL links are not clickable in rendered view
**Solution**: This indicates the Gist filename does not have the `.md` extension. Correct the filename per the first troubleshooting item — all `[text](url)` Markdown links render as clickable blue links in GitHub Gist when the filename has `.md`.

**Problem**: YAML frontmatter renders as visible `---` text breaking into the body
**Solution**: This is a Gist rendering edge case for some frontmatter formats. It does not affect readability of the document body — recipients focus on the content sections below it. No action required unless the frontmatter content itself is needed by a specific recipient.

**Emergency fallback (Gist creation fails or takes too long)**:
Send Wave 1 emails with the source document attached as a `.md` file rather than a Gist URL. This is suboptimal — not all email clients render Markdown — but functional. Note in DOMAIN_48_DISTRIBUTION_SEND_LOG_TEMPLATE.md. Create the Gist afterward for Wave 2 distribution.
