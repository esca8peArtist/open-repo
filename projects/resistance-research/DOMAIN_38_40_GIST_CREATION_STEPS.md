## Troubleshooting (Both Domains)

**Problem**: Gist shows as plain text (no formatting)
**Solution**: Verify filename ends in `.md`. Edit the Gist (pencil icon on the Gist page), correct the filename, save. GitHub Gist re-renders automatically after filename correction.

**Problem**: Gist text appears truncated (Sources section missing)
**Solution**: Edit the Gist, scroll to the bottom of the editor, verify the full source section is present. If truncated, paste the missing text at the end and save. Domain 38 ends with Source Index item 47/48; Domain 40 ends with the 47th footnote.

**Problem**: URL requires login when tested in incognito
**Solution**: Verify the Gist was created as "Secret" or "Public" (not accidentally saved as a draft). Delete and recreate with explicit "Secret" selection if needed. Secret Gists are publicly accessible to anyone with the URL.

**Problem**: Source URL links are not clickable in rendered view
**Solution**: This indicates the Gist filename does not have the `.md` extension. Correct the filename per above — all `[text](url)` Markdown links render as clickable blue links in GitHub Gist when the filename has `.md`.

**Problem**: YAML frontmatter renders oddly (raw `---` text visible)
**Solution**: This is a Gist rendering edge case for some frontmatter formats. It does not affect readability of the document body. The frontmatter is metadata; recipients focus on the content sections below it. No action required unless the frontmatter content itself is needed by a specific recipient.

**Emergency fallback (no time to create Gist)**: Send Tier A emails with the source document as an attachment (`.md` file) rather than a Gist URL. This is suboptimal (not all email clients render Markdown) but functional. Note in `DISTRIBUTION_EXECUTION_LOG.md`. Create the Gist afterward for Tier B/C/D distribution.
