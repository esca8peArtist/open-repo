## Step 10 — Troubleshooting

**Problem**: Document renders as plain text (no Markdown formatting)  
**Solution**: Verify the filename ends in `.md`. Edit the Gist (pencil icon on Gist page), correct the filename to `domain-40-surveillance-capitalism-electoral-manipulation.md`, save. GitHub re-renders automatically.

**Problem**: Document appears truncated — Sources section missing  
**Solution**: Edit the Gist, scroll to the bottom of the editor, verify all 47 sources are present. If not, paste the missing text at the end and save.

**Problem**: URL requires login when tested in incognito  
**Solution**: Verify the Gist is set to "Secret" or "Public" (not "Private to you" — GitHub Gist has no private setting). If the issue persists, delete the Gist and recreate with explicit "Secret" selection.

**Problem**: The YAML frontmatter renders oddly (as raw `---` text)  
**Solution**: This is a Gist rendering edge case. It does not affect readability of the document body. The frontmatter is metadata; recipients will focus on the content sections below it.

**Problem**: No time to create Gist before July 15 — emergency fallback  
**Solution**: Send Tier A emails with the source document as an attachment (`.md` file) rather than a Gist URL. This is suboptimal (not all email clients render Markdown) but is functional. Note in DISTRIBUTION_EXECUTION_LOG.md. Create the Gist afterward for Tier B/C/D distribution.
